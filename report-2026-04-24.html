進度摘要

進行 Jetson Nano 邊緣運算設備的環境建置與部署，記錄 SD 卡存儲不穩定及 MediaPipe 與 OpenCV 版本衝突等問題，並規劃對應解決方案，以確保後續步態分析模型能正常取得即時影像。

1. Jetson Nano 基本介紹

Jetson Nano 是 NVIDIA 推出的嵌入式 AI 開發板，具備以下三項核心優勢，適合本專題的邊緣端推論需求：

邊緣運算
可在本地端直接處理感測器數據，不需上傳雲端，降低延遲並提高資料隱私性。
模型訓練與推論
支援 TensorFlow、PyTorch 等框架，可執行 OpenGait 或 VLLM 輕量化模型。
高效能功耗比
在低功耗下提供充足的 CUDA 核心運算能力，適合長時間連續部署。

2. 開發流程整理

  1. 環境初始化
    安裝 JetPack SDK 並燒錄系統鏡像至 SD 卡。
  2. 基礎庫安裝
    配置 Python 虛擬環境，安裝必要的開發依賴項。
  3. 模型轉換
    將預訓練模型轉換為 TensorRT 格式，以優化推論速度。
  4. 視覺框架整合
    引入電腦視覺庫進行影像串流處理。
  5. 系統整合與測試
    進行端到端測試，目前執行至此階段，遭遇以下問題待解決。

3. 目前遭遇問題與待辦事項

問題一：硬體層面 — 存儲介質不穩定

問題描述：記憶卡（SD Card）出現異常，導致系統在執行高負載任務時發生當機或讀取錯誤。

預計對策：

  • 檢查 SD 卡讀寫速度等級（建議使用 Class 10 / UHS-1 以上規格）。
  • 嘗試重新燒錄系統鏡像，或更換更穩定的工業級記憶卡。
  • 考慮掛載外部 SSD 作為系統開機碟，以提升整體穩定性。
問題二：軟體環境 — 函式庫版本衝突 (Dependency Conflict)

問題描述：安裝 MediaPipe 後，其依賴項覆蓋了 Jetson Nano 內建、經 NVIDIA 優化過的 OpenCV 版本。

影響：系統無法正確調用 CSI 鏡頭（CSI Camera 需特定含 GStreamer 支援的 OpenCV 版本）。

預計對策：

  • 移除目前衝突的 OpenCV 版本。
  • 重新編譯或連結回支援 NVIDIA 加速的 OpenCV 版本（需包含 WITH_GSTREAMER=ON 編譯標誌）。
  • 使用 Docker 容器化開發環境，將 MediaPipe 與系統層級的影像驅動庫隔離，避免版本干擾。
本週優先任務：優先解決 CSI 鏡頭驅動問題，確保後續步態分析模型能正常取得即時影像串流，再推進模型推論整合。
