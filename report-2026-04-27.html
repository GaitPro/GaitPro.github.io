進度摘要

成功解決上週 Jetson Nano 的 CSI 鏡頭驅動問題。透過 GStreamer Pipeline 驗證硬體連接正常，並完成 Python / OpenCV 程式整合，最終成功於螢幕上即時顯示鏡頭畫面，為後續步態分析模型的影像輸入奠定基礎。

一、硬體裝置檢查

確認系統是否正確識別 CSI 鏡頭裝置，執行以下指令：

ls /dev/video*
裝置確認：系統回傳 /dev/video0，代表 CSI 鏡頭硬體已成功掛載。
裝置名稱：vi-output, imx219 6-0010  ｜  平台位置：platform:54080000.vi:0
圖 1：終端機確認 /dev/video0 裝置路徑，並輸入 GStreamer Pipeline 測試指令

二、GStreamer 管道（Pipeline）測試

在進入程式開發前，先以指令介面直接測試 nvarguscamerasrc 插件能否正常驅動鏡頭：

gst-launch-1.0 nvarguscamerasrc ! \
  'video/x-raw(memory:NVMM), width=640, height=480, framerate=30/1, format=NV12' ! \
  nvvidconv flip-method=0 ! \
  nvoverlaysink -e
GST_ARGUS: Setup Complete — 驅動程式與鏡頭握手成功。
Producer has connected; continuing. — 影像串流開始傳輸。
系統輸出詳細鏡頭參數，確認支援 1280×720 @ 120fps 等多種模式。
圖 2：終端機顯示 GST_ARGUS 完整啟動過程，最終輸出 Done Success 確認串流結束正常

三、Python 與 OpenCV 程式整合

將測試成功的 GStreamer 語法封裝進 Python 函式，以便後續進行影像處理。定義 gstreamer_pipeline 函式，將硬體加速格式轉換為 OpenCV 易於處理的 BGR 格式：

import cv2

def gstreamer_pipeline(width=1280, height=720, fps=30):
  return (
    f"nvarguscamerasrc ! "
    f"video/x-raw(memory:NVMM), width={width}, height={height}, "
    f"format=NV12, framerate={fps}/1 ! "
    f"nvvidconv ! video/x-raw, format=BGRx ! "
    f"videoconvert ! video/x-raw, format=BGR ! appsink"
  )

cap = cv2.VideoCapture(gstreamer_pipeline(), cv2.CAP_GSTREAMER)
圖 3：在 Jetson Nano 上以 Vim 編輯 Python 腳本，實作 GStreamer Pipeline 並呼叫 cv2.VideoCapture

四、最終測試結果

成功執行 Python 腳本，畫面順利顯示於螢幕上，延遲極低且符合硬體加速預期。
